## Key Features

- **Monorepo Structure**: Using Turborepo for efficient monorepo management
- **Shared UI Components**: Shadcn/UI components available across all applications
- **Type Safety**: Full TypeScript support throughout the project
- **Consistent Styling**: Tailwind CSS configuration shared across packages

## Project Organization

### Apps Directory

The `apps` directory contains all your applications:

- `web`: Main web application
- `docs`: Documentation site

### Packages Directory

The `packages` directory contains shared packages:

- `ui`: Shared UI components built with Shadcn/UI

## Using Shadcn/UI Components

You can import components from the shared UI package:


<Step copyCommand={false}>
```tsx
import { Button } from "@/components/ui/button"
import { Card } from "@/components/ui/card"

export default function MyComponent() {
  return (
    <Card>
      <Button>Click me</Button>
    </Card>
  )
}
```
</Step>

## Adding New Components

To add a new Shadcn/UI component:

1. Add it to the UI package:

<Step copyCommand={false}>

```bash
cd packages/ui
pnpm dlx shadcn-ui@latest add button
```
</Step>

2. Export it from the package:

<Step copyCommand={false}>
```tsx
// packages/ui/index.tsx
export * from './components/button'
```
</Step>


3. Use it in your apps:

<Step copyCommand={false}>
```tsx
import { Button } from '@repo/ui'
```
</Step>

## Development Workflow

1. Make changes to shared components in `packages/ui`
2. Changes will automatically reflect in all apps using the components
3. Use `pnpm run dev` to run all apps in development mode
4. Use `pnpm run build` to build all apps and packages

## Configuration Files

- `turbo.json`: Defines the task pipeline and dependencies
- `tailwind.config.ts`: Shared Tailwind configuration
- `tsconfig.json`: Base TypeScript configuration
